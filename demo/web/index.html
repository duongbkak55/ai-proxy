    Diễn giải

    1. t=00s Dev set ANTHROPIC_BASE_URL vào proxy local + chạy claude như bình thường — chỉ 1 dòng config.
    2. t=08–13s Dev gõ prompt có email khách (alice@example.com), số ĐT (0912345678), codename (ProjectAlpha), package nội bộ (@vng/auth), tên bảng DB (vng_user_payments). Plaintext đầy đủ.
    3. t=13.6s Proxy nhận POST /v1/messages → auth pass → DLP scan.
    4. t=13.9–14.1s 5 match: 2 regex (email, phone) + 3 dictionary (codename, pkg, db table) → vault issue 5 token.
    5. t=14.4s Proxy forward request đã redact sang api.anthropic.com — model chỉ thấy EMAIL_01, PHONE_VN_01, CODENAME_01, PKG_01, DBTABLE_01.
    6. t=16–24s Stream SSE về: mỗi chunk có token → proxy detokenize on-the-fly → dev terminal hiện plaintext lại.
    7. t=24.8s Audit log fsync: hash chain entry với summary (5 match, 7 detok events, blocked=false).

    Takeaway: developer experience không đổi (Claude Code hiện plaintext alice@example.com như bình thường), nhưng api.anthropic.com KHÔNG BAO GIỜ thấy plaintext.

    Pipeline thật trong production có thêm: SQL AST lane (fenced ```sql), Source-code AST lane (fenced ```ts/py/java), Auth lane (bearer token + rate-limit), Allowlist (banned tools, SSRF). demo bundle: /demo/web/bundle.js
